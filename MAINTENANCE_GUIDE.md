## 添加新页面

### 详细步骤指南

#### 步骤1: 规划新页面
1. **确定页面用途和目标受众**
   - 明确页面功能和内容类型
   - 确定是否需要多语言版本
   - 评估与现有页面的关系

2. **选择合适的模板**
   - 参考现有页面结构
   - 确定所需的CSS和JS依赖
   - 规划页面布局和组件

#### 步骤2: 创建页面文件
1. **英文版本页面**
   ```bash
   # 在 en/ 目录创建新页面
   # 文件名使用小写字母和连字符
   touch en/new-page.html
   ```

2. **中文版本页面**
   ```bash
   # 在 zh/ 目录创建对应页面
   touch zh/new-page.html
   ```

#### 步骤3: 编写页面内容
1. **使用标准HTML结构**
   ```html
   <!DOCTYPE HTML>
   <html>
   <head>
       <title>页面标题 - Qingguang Zheng</title>
       <meta charset="utf-8" />
       <meta name="viewport" content="width=device-width, initial-scale=1, user-scalable=no" />
       <link rel="stylesheet" href="../assets/css/main.css" />
   </head>
   <body class="is-preload">
       <!-- 页面内容 -->
   </body>
   </html>
   ```

2. **添加导航菜单**
   - 在现有页面的导航部分添加新页面链接
   - 确保英文和中文版本都有对应链接
   - 保持导航结构一致性

#### 步骤4: 更新语言切换功能
1. **修改语言切换逻辑**
   - 在相关JavaScript文件中添加新页面的语言映射
   - 确保用户可以在新页面间切换语言

2. **测试语言切换**
   - 验证从新页面可以正确切换到对应语言版本
   - 检查URL路径映射是否正确

## 内容更新工作流程

### 单页面内容更新

#### 准备工作
1. **备份原始内容**
   ```bash
   # 创建备份文件
   cp en/research.html en/research.html.backup
   cp zh/research.html zh/research.html.backup
   ```

2. **启动本地服务器**
   ```bash
   # Windows
   start-local-server.bat
   
   # Linux/Mac
   ./start-local-server.sh
   
   # 或直接运行Python服务器
   python local-server.py
   ```

#### 更新流程
1. **编辑页面内容**
   - 使用文本编辑器或IDE打开目标HTML文件
   - 修改页面内容，保持HTML结构完整
   - 注意保持样式类名和ID的一致性

2. **实时预览和调试**
   - 在浏览器中访问 `http://localhost:8000`
   - 刷新页面查看更改效果
   - 使用浏览器开发者工具调试样式问题

3. **多设备测试**
   - 测试桌面端显示效果
   - 使用浏览器响应式模式测试移动端
   - 检查不同屏幕尺寸下的布局

#### 质量检查
1. **内容检查**
   - [ ] 文字内容准确无误
   - [ ] 链接地址正确有效
   - [ ] 图片显示正常
   - [ ] 格式和样式一致

2. **技术检查**
   - [ ] HTML语法正确
   - [ ] CSS样式正常应用
   - [ ] JavaScript功能正常
   - [ ] 页面加载速度合理

### 批量内容更新

#### 多页面同步更新
1. **制定更新计划**
   - 列出需要更新的页面清单
   - 确定更新的优先级和顺序
   - 预估所需时间和资源

2. **统一更新流程**
   ```bash
   # 批量备份文件
   for file in en/*.html; do
       cp "$file" "$file.backup"
   done
   
   for file in zh/*.html; do
       cp "$file" "$file.backup"
   done
   ```

3. **逐页验证**
   - 按计划顺序更新每个页面
   - 完成一个页面后立即测试
   - 记录更新过程中的问题和解决方案

## 多语言内容同步

### 同步策略

#### 内容对应关系
```
英文版本 (en/)          中文版本 (zh/)
├── index.html    ←→   ├── index.html
├── research.html ←→   ├── research.html  
├── skills.html   ←→   ├── skills.html
├── personal.html ←→   ├── personal.html
├── QAs.html      ←→   ├── (无对应页面)
└── contact.html  ←→   ├── contact.html
                       └── blog.html (中文独有)
```

#### 同步工作流程
1. **确定主要版本**
   - 通常以英文版本为主进行更新
   - 重要内容优先更新英文版本
   - 中文版本根据需要进行相应调整

2. **内容翻译和本地化**
   - 不是简单的逐字翻译
   - 考虑文化差异和表达习惯
   - 保持核心信息的一致性

3. **版本控制**
   - 使用Git跟踪多语言版本的变化
   - 为重要更新创建分支
   - 记录翻译和本地化的决策

### 同步操作指南

#### 新增内容同步
1. **英文版本更新后**
   ```markdown
   步骤：
   1. 完成英文版本内容更新
   2. 测试英文版本功能正常
   3. 翻译内容到中文版本
   4. 调整中文版本的表达方式
   5. 测试中文版本功能
   6. 验证语言切换功能
   ```

2. **中文独有内容**
   ```markdown
   对于中文版本独有的内容（如blog.html）：
   1. 确保内容符合整体风格
   2. 保持导航结构的一致性
   3. 考虑是否需要英文版本对应页面
   ```

#### 链接和导航同步
1. **内部链接管理**
   - 确保两个语言版本的内部链接都正确
   - 更新导航菜单时同步两个版本
   - 检查语言切换链接的准确性

2. **外部链接处理**
   - 优先使用英文版本的外部链接
   - 如有中文版本的对应资源，提供中文链接
   - 确保链接在两个版本中都有效

## 常见问题解决方案

### 页面显示问题

#### 问题1: 页面样式丢失
**症状**: 页面显示为纯HTML，没有CSS样式

**可能原因**:
- CSS文件路径错误
- CSS文件被移动或删除
- 服务器配置问题

**解决方案**:
```html
<!-- 检查CSS引用路径 -->
<link rel="stylesheet" href="../assets/css/main.css" />

<!-- 确保路径相对于当前HTML文件位置正确 -->
<!-- 从 en/index.html 到 assets/css/main.css 应该是 ../assets/css/main.css -->
```

#### 问题2: 图片无法显示
**症状**: 页面中图片显示为破损图标

**可能原因**:
- 图片文件路径错误
- 图片文件不存在
- 图片文件权限问题

**解决方案**:
```html
<!-- 检查图片路径 -->
<img src="../images/pic01.jpg" alt="描述文字" />

<!-- 验证图片文件是否存在 -->
<!-- 检查文件名大小写是否匹配 -->
```

### 导航和链接问题

#### 问题3: 内部链接失效
**症状**: 点击导航链接出现404错误

**解决方案**:
1. **检查文件路径**
   ```html
   <!-- 确保链接指向正确的文件 -->
   <a href="research.html">Research</a>  <!-- 同目录 -->
   <a href="../zh/research.html">中文版</a>  <!-- 跨目录 -->
   ```

2. **验证文件存在**
   ```bash
   # 检查目标文件是否存在
   ls -la en/research.html
   ls -la zh/research.html
   ```

#### 问题4: 语言切换失效
**症状**: 点击语言切换链接没有反应或跳转错误

**解决方案**:
1. **检查JavaScript文件**
   - 确保语言切换脚本正确加载
   - 验证脚本中的页面映射关系

2. **更新页面映射**
   ```javascript
   // 确保新页面包含在语言映射中
   const pageMapping = {
       'en/index.html': 'zh/index.html',
       'en/research.html': 'zh/research.html',
       // 添加新页面映射
   };
   ```

### 本地服务器问题

#### 问题5: 本地服务器无法启动
**症状**: 运行启动脚本后服务器没有响应

**解决方案**:
1. **检查Python环境**
   ```bash
   # 验证Python是否安装
   python --version
   # 或
   python3 --version
   ```

2. **检查端口占用**
   ```bash
   # Windows
   netstat -an | findstr :8000
   
   # Linux/Mac  
   lsof -i :8000
   ```

3. **手动启动服务器**
   ```bash
   # 直接运行Python服务器
   python local-server.py
   
   # 或使用Python内置服务器
   python -m http.server 8000
   ```

#### 问题6: 页面更新不显示
**症状**: 修改文件后浏览器显示旧内容

**解决方案**:
1. **清除浏览器缓存**
   - 按 Ctrl+F5 强制刷新
   - 或在开发者工具中禁用缓存

2. **检查文件保存**
   - 确认文件已正确保存
   - 检查文件修改时间

## 故障排除方法

### 系统性问题诊断

#### 诊断流程
1. **确定问题范围**
   - 是单个页面问题还是全站问题？
   - 是特定浏览器问题还是通用问题？
   - 是本地环境问题还是代码问题？

2. **收集错误信息**
   - 查看浏览器控制台错误信息
   - 检查网络请求状态
   - 记录具体的错误症状

3. **逐步排查**
   - 从最简单的可能原因开始
   - 逐一验证每个可能的问题点
   - 记录排查过程和结果

### 常用调试工具

#### 浏览器开发者工具
```
Chrome/Edge/Firefox 开发者工具使用：
1. 按 F12 打开开发者工具
2. Console 标签页：查看JavaScript错误
3. Network 标签页：检查资源加载状态
4. Elements 标签页：检查HTML结构和CSS样式
5. Sources 标签页：调试JavaScript代码
```

#### 文件验证工具
```bash
# 检查HTML语法
# 可以使用在线HTML验证器：https://validator.w3.org/

# 检查文件编码
file -I filename.html

# 检查文件权限
ls -la filename.html
```

### 应急恢复程序

#### 快速回滚
1. **使用备份文件**
   ```bash
   # 恢复单个文件
   cp filename.html.backup filename.html
   
   # 批量恢复
   for file in *.backup; do
       mv "$file" "${file%.backup}"
   done
   ```

2. **使用Git版本控制**
   ```bash
   # 查看最近的提交
   git log --oneline -5
   
   # 回滚到特定提交
   git checkout [commit-hash] -- filename.html
   
   # 撤销最近的更改
   git checkout HEAD -- filename.html
   ```

#### 紧急联系和支持
```
当遇到无法解决的问题时：
1. 记录详细的错误信息和重现步骤
2. 保存当前状态的截图
3. 备份当前文件状态
4. 寻求技术支持或查阅相关文档
```

## 总结

本维护指南涵盖了个人主页项目的主要维护操作，从日常内容更新到复杂的多语言同步，从常见问题解决到系统性故障排除。

### 关键要点
1. **预防胜于治疗**: 定期备份和检查可以避免大多数问题
2. **渐进式更新**: 小步快跑，每次更改后立即测试
3. **文档记录**: 记录操作过程和问题解决方案
4. **多设备测试**: 确保在不同设备和浏览器上都能正常工作

### 持续改进
- 根据实际使用情况更新本指南
- 收集和分析常见问题，完善解决方案
- 关注新技术和最佳实践，适时更新维护流程

遵循本指南的操作流程和检查清单，可以确保项目的稳定性和可维护性，为用户提供良好的访问体验。
